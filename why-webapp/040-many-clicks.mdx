## 4. It takes many clicks to get feedback

Most CI/CD systems use a "checks API" to give a pass/fail metric for a pull request.

For example, they might tell you whether a build failed or if a unit test passed.

However, webapps have many numerical metrics like page speed that aren't necessarily pass/fail.

**How webapp.io solves this problem:**

Webapp.io allows you to embed customizable widgets into your pull requests so that reviewers can see details about a specific change at a glance.
